# Тесты

100% покрытие — не самоцель, но тесты дают уверенность двигаться быстро и не бояться менять код.

## Запуск

```
.venv/bin/python -m pytest tests/ -v
```

## Слои
- **Unit** (`tests/`) — чистая логика без обращения к Google Sheets (например `core/inventory.with_balances`). Sheets-функции (`core/sheets.py`) не мокаются — тестируем то, что можно проверить без сети.

## Конвенции
- Файл `tests/test_<module>.py` на модуль из `core/`.
- Имя теста описывает поведение: `test_<функция>_<сценарий>`.
- Проверяем реальное значение (например `== 7.25`), не `is not None`.
