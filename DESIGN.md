# Design System — oinarri

## Product Context
- **What this is:** внутренний инструмент учёта сырья/производства/продаж для мыловарни/косметической мастерской
- **Кто пользуется:** Founder, Worker (часто без техбэкграунда — "любая обезьяна должна справиться"), Developer
- **Индустрия:** ремесленное производство (мыло, свечи, косметика)
- **Тип проекта:** внутренний веб-инструмент (dashboard + формы + таблицы), не публичный продукт

## Статус внедрения (2026-07-17)

Система внедрена **частично, осознанно, поэтапно** — не единомоментный редизайн всего фронта:

- ✅ **Sidebar/Layout** (`Sidebar.tsx`, `Layout.tsx`, `Brand.tsx`) — премиум-тема применена, видна на КАЖДОЙ странице (Sidebar общий компонент)
- ✅ **Дашборд** (`DashboardPage.tsx`, `components/widgets/*`) — полностью на новой системе, flagship-экран
- ⬜ **Остальные 8 страниц** (Компоненты, Производство, Рецепты, Отгрузка, Продукты, Сотрудники, Техпанель, Видеонаблюдение) — пока на старой светлой теме (cream/ink/terracotta), не тронуты. Следующая фаза.
- Логин (`LoginPage.tsx`) — не тронут, у него уже была своя приличная тёмная/светлая тема (blob-анимация, backdrop-blur) — не то, из-за чего Founder сказала "no wow"

**Почему частично:** до этой правки в приложении не было app-wide dark mode вообще (тумблер темы был только на логине) — полный редизайн всех 11 страниц + всех модалок означал большой рискованный дифф прямо перед code-review/security-review. Founder подтвердила такой поэтапный план (см. CLAUDE.md → "Design System").

## Aesthetic Direction
- **Направление:** Refined Industrial — тёмная, точная типографика, много воздуха вокруг плотных данных, один сдержанный акцент
- **Уровень декора:** intentional — grain-текстура на фоне, hero-цифры с золотым свечением, но не expressive/декоративно
- **Настроение:** серьёзный премиум-инструмент, тёплый (не холодное техно-стекло), с отсылкой к ремеслу через золото/воск
- **Референсы:** built-in дизайн-знания (без внешнего рисёрча — так решил пользователь), в духе Linear/Arc по дисциплине, но тёплая палитра вместо синей

## Typography
- **Display/заголовки страниц:** Fraunces italic 500 — единственное место с serif, сигнал мастерства/премиума
- **Body/интерфейс:** системный sans-serif (не менялся в core-проходе — глобальный `body` font-family трогать не стали, чтобы не задеть 8 нетронутых страниц)
- **Data/графики (только Дашборд):** Geist Mono / IBM Plex Mono для табличных данных
- **Правило:** Fraunces НИКОГДА не в теле/таблицах, только заголовки h1 уровня страницы — иначе система разваливается
- **Загрузка:** Google Fonts, `index.html` (`Fraunces` + `Geist Mono`, вес 500/400-600, `display=swap`)

## Color

Токены — `frontend/src/index.css`, Tailwind `@theme` блок, префикс `premium-*` (аддитивно, старые `cream/ink/terracotta` НЕ удалены и НЕ переопределены — используются 8 нетронутыми страницами).

| Токен | Hex | Назначение |
|---|---|---|
| `--color-premium-bg` | `#14120F` | фон страницы (тёплый почти-чёрный, не чистый) |
| `--color-premium-surface` | `#1D1A16` | карточки/панели |
| `--color-premium-surface-2` | `#262219` | hover-фон, вложенные поверхности |
| `--color-premium-border` | `#332D24` | границы |
| `--color-premium-text` | `#F3EEE4` | основной текст |
| `--color-premium-text-muted` | `#A99C89` | вторичный текст |
| `--color-premium-gold` | `#C6A15B` | единственный акцент — активная навигация, кнопки, hover-свечение |
| `--color-premium-gold-hi` | `#DDBB7C` | hover/активное состояние акцента |

Графики (recharts, только виджеты Дашборда): `CHROME_DARK` в `frontend/src/lib/vizColors.ts` — тёмный вариант chrome-цветов (сетка/оси/тултип). `CATEGORICAL`/`STATUS` (сами цвета данных, CVD-защищённая палитра) **не менялись** — они одинаково легитимны на любом фоне, порядок слотов трогать нельзя (см. skill `dataviz`).

- **Dark mode:** система dark-first, отдельного light-варианта для premium-токенов пока нет (не требовалось — тронутые экраны все тёмные)

## Spacing
- **База:** существующая Tailwind-шкала, ничего нового не вводили
- **Плотность:** comfortable — обоснование то же, что у Founder в CLAUDE.md ("любая обезьяна должна справиться"), тесные таблицы этому мешают

## Layout
- **Подход:** без изменений — Sidebar + content, как было (grid-disciplined)
- **Border radius:** без изменений (Tailwind `rounded-lg`/`rounded-xl`, как в остальном приложении)

## Motion
- **Подход:** intentional
- `.premium-card:hover` — подъём на 2px + золотая рамка (`frontend/src/index.css`)
- `.premium-stat:hover` — золотое свечение (`box-shadow`) + подъём
- `.premium-stat::after` — гигантская полупрозрачная цифра-водяной знак позади hero-числа (Fraunces italic, opacity .07)
- `.premium-grain` — SVG grain-текстура поверх фона (opacity .05, `mix-blend-mode: overlay`), убирает "плоский hex" эффект
- Все анимации уважают `prefers-reduced-motion` (отключаются)

## Что дало "вау"-эффект (после критики второй агент-модели, 2026-07-17)
Первая версия была "enterprise-SaaS-шаблон с красивым шрифтом" — вся смелость свелась к одному градиенту. Добавлено точечно:
1. Grain-текстура на фоне — оптическая глубина вместо плоского цвета
2. Hero-цифра дашборда — крупная Fraunces italic с золотым свечением и водяным знаком позади
3. Hover на строках таблиц/списков — золотая полоса слева при наведении
4. Карточки — подъём + золотая рамка при hover
5. "Восковая печать" на статусе успеха — отсылка к ремеслу мыловарни (заведено как паттерн, ещё не применено ни на одной живой странице — было только в HTML-мокапе)
